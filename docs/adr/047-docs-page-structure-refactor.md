# ADR 047: DocsPageをデータ駆動・セクション分割で再構成する

## ステータス

Accepted

## コンテキスト

- `DocsPage.tsx` が巨大化し、UI構造・データ・レンダリングロジックが単一ファイルに混在している
- セクション追加や文言修正のたびに広範な差分が発生し、レビューと変更確認の負荷が高い
- サイドバーと本文のID整合・スクロール制御が散在しており、変更時に整合性を崩しやすい

## 決定

- ドキュメントページを「データ定義」「UIコンポーネント」「セクションレンダリング」に分割する
- 章構成とサブセクションIDは `docs/data.tsx` に集約し、参照元を統一する
- 既存UIの見た目と動作（表示順、スクロール、モーション）は維持する

## 理由

- 可読性: ドキュメント構造とUIロジックを分離し、変更の焦点を明確にする
- 変更容易性: 文章追加やセクション追加がデータ側の変更に集中する
- 安全性: IDの単一ソース化により、サイドバーと本文の不整合を減らす

## リスクと対応

| リスク | 対応 |
|--------|------|
| 分割によりファイル数が増える | ドキュメント専用ディレクトリに集約し探索性を維持する |
| データ定義とUIの依存が増える | 型定義を導入して構造の整合性を保証する |
| セクションIDの不整合でスクロール不具合が出る | サブセクションIDとメインキーをデータから生成して一元管理する |
