# ロードマップ

## ビジョン

**最終目標**: 法人向けCASB（Cloud Access Security Broker）
- AIサービスの利用状況を可視化し、プライバシーリスクを管理
- 端末完結型（サーバーレス）のブラウザ拡張機能
- プライバシー保護（データは端末に留まる）
- 「検出→可視化→ポリシー生成」のフロー

## Phase 1: MVP強化（現在）

**目標**: 個人ユーザー向け価値検証

### 実装済み
- サービス検出（ログインページ、認証）
- CSP監査
- AIプロンプト監視（基本）

### 未実装
| 機能 | 説明 | 優先度 |
|-----|------|-------|
| PII検出 | プロンプト内の個人情報（メール、電話番号等）検出 | 高 |
| リスクスコアリング | AIプロンプトの機密性レベル自動評価 | 高 |
| エクスポート機能 | 監査ログのCSV/JSON出力 | 中 |

## Phase 2: 可視化強化

**目標**: データドリブンな意思決定支援

| 機能 | 説明 | 優先度 |
|-----|------|-------|
| ダッシュボード改善 | 時系列分析、トレンド表示 | 高 |
| Shadow AI検出強化 | 未知のAIサービスの自動分類 | 高 |
| 拡張機能リスク監視 | 危険なブラウザ拡張機能の検出 | 中 |
| Identity Security | パスワード再利用、弱いパスワード検出 | 中 |

## Phase 3: ポリシーエンジン

**目標**: 検出から保護へ

| 機能 | 説明 | 優先度 |
|-----|------|-------|
| アラート機能 | 高リスク検出時の通知 | 高 |
| ポリシー自動生成 | 可視化データからルール提案 | 高 |
| DLPルール | APIキー、クレカ番号等のパターン検出 | 高 |
| ローカルブロック | オプショナル、ユーザー同意ベース | 低 |

## Phase 4: エンタープライズ対応

**目標**: 法人向けCASBへの進化

| 機能 | 説明 | 優先度 |
|-----|------|-------|
| オプショナルサーバー連携 | 組織での一元管理 | 高 |
| SSO統合 | SAML/OIDC対応 | 高 |
| コンプライアンスレポート | HIPAA、CCPA対応 | 中 |
| マルチブラウザ対応 | Firefox、Safari、Edge | 中 |

## Phase 5: インテリジェンス

**目標**: 高度な分析機能

| 機能 | 説明 | 優先度 |
|-----|------|-------|
| AI利用パターン分析 | 組織全体の傾向 | 中 |
| リスクベンチマーク | 業界平均との比較 | 低 |
| 予測分析 | リスク予兆の検出 | 低 |
| 軽量脅威DB | ユーザー同意ベースのバンドル | 低 |

## リスクと対策

| リスク | 対策 |
|-------|------|
| ローカル完結の限界 | Phase 4でオプショナルサーバー連携 |
| Manifest V3制限 | WXT採用で対応済み |
| 競合の追随 | OSSとプライバシーで差別化維持 |
| サプライチェーン攻撃 | コード署名、リリース監査強化 |
